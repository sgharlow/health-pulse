# HealthPulse AI — Prompt Opinion Demo Guide

> Step-by-step checklist for recording the hackathon demo video.
> The video MUST show the project working INSIDE Prompt Opinion platform.
> Maximum 3 minutes. Upload to YouTube as Public or Unlisted.

---

## Pre-Recording Checklist

Do these BEFORE you start recording:

- [ ] Open https://app.promptopinion.ai in Chrome at 1920x1080
- [ ] Navigate to **Marketplace Studio > MCP Servers**
- [ ] Confirm **HealthPulse AI** has the **Published** toggle **ON**
- [ ] If toggle is OFF, turn it ON and wait 30 seconds
- [ ] Navigate to **Launchpad** and select **HealthPulse AI** agent
- [ ] Enable "Show Tool Calls" toggle (if available) so judges can see which MCP tools are invoked
- [ ] Send a warmup query: `test` — wait for it to respond (warms up Railway cold start)
- [ ] Close all other tabs, notifications, and anything showing credentials
- [ ] Open your screen recorder (OBS, Loom, or Win+G Game Bar)
- [ ] Set recording to 1920x1080
- [ ] Have this script visible on a second monitor or printed

## Recording Script (2:55)

### [0:00 - 0:15] Introduction

**Do:** Start recording. Show the Prompt Opinion Launchpad with HealthPulse AI selected.

**Say:**
"HealthPulse AI is a healthcare performance intelligence agent with 11 MCP tools, analyzing over 233,000 rows of real CMS hospital quality data across 5,400 US hospitals. Let me show you what it can do."

---

### [0:15 - 0:45] Query 1: Quality Anomaly Detection

**Type in Prompt Opinion:**
```
Show me quality anomalies in California hospitals
```

**Wait** for the response. The agent will call `quality_monitor`.

**Say (while waiting/after response):**
"I asked for quality anomalies in California. The agent called the quality_monitor tool, which uses Z-score analysis across all facilities. It found 110 critical anomalies — facilities that look normal on absolute numbers but are 3 or more standard deviations from their peers. That kind of statistical pattern detection is impossible in a traditional dashboard."

**Point out:** Specific facility names, Z-scores, severity levels in the response.

---

### [0:45 - 1:15] Query 2: Cross-Cutting Multi-Factor Analysis

**Type:**
```
Find hospitals in Florida with multiple compounding concerns
```

**Wait** for response. Agent calls `cross_cutting_analysis`.

**Say:**
"This is the tool that shows AI's real value. The cross-cutting analysis checks six dimensions simultaneously: quality measures, readmission rates, health equity, patient experience, cost efficiency, and star ratings. In Florida, 94 of 222 hospitals have two or more compounding risk factors. These are systemic patterns that no single-metric dashboard can surface."

**Point out:** A facility with 3+ concerns, the specific concern types listed.

---

### [1:15 - 1:40] Query 3: Health Equity Disparities

**Type:**
```
Check for equity disparities in Texas hospitals
```

**Wait** for response. Agent calls `equity_detector`.

**Say:**
"The equity detector correlates hospital quality outcomes with the CDC Social Vulnerability Index at the county level. Nationally, there's a 0.77-star rating gap between hospitals serving high-vulnerability communities and those in affluent areas. That's a systemic disparity affecting 1,700 facilities and millions of patients — computed with a single tool call against real public data."

---

### [1:40 - 2:05] Query 4: Patient Risk Profile (FHIR)

**Type:**
```
Show me the patient risk profile for facility 050454
```

**Wait** for response. Agent calls `patient_risk_profile`.

**Say:**
"Now we bridge from population analytics to individual patients. The patient risk profile tool uses synthetic FHIR data — 100 patients generated by Synthea — to demonstrate how the same MCP server can drill down to clinical-level data when connected to an EHR. SHARP headers propagate the FHIR context on every request. This is the architecture that real health systems are building today."

**Point out:** Risk distribution (high/medium/low), specific patient conditions.

---

### [2:05 - 2:30] Query 5: Cost Efficiency

**Type:**
```
Which California hospitals are overspending on Medicare?
```

**Wait** for response. Agent calls `cost_efficiency`.

**Say:**
"The cost efficiency tool analyzes Medicare spending per beneficiary against quality outcomes. In California, 52 percent of overspending hospitals also have below-average star ratings. Higher spending does not mean better care. That's the kind of finding that directly informs value-based purchasing decisions."

**Point out:** Spending ratios, cost-quality correlation numbers.

---

### [2:30 - 2:55] Closing

**Say:**
"HealthPulse AI: 11 MCP tools spanning quality, readmissions, equity, patient experience, cost efficiency, and patient-level FHIR analysis. 233,000 rows of real CMS data. 100 synthetic FHIR patients. ## Troubleshooting

**If Prompt Opinion responds slowly:**
- The Railway server may cold-start after inactivity. Send a warmup query first.
- If still slow, you can speed up the video 1.5x during tool execution in post-editing.

**If a tool fails:**
- Check that the MCP server is running: visit `https://health-pulse-mcp-production.up.railway.app/mcp` in browser (should show "Method Not Allowed" for GET).
- Try a simpler query first: `What is the average star rating across all hospitals?`

- Go to Marketplace Studio > MCP Servers and verify Published is ON.
